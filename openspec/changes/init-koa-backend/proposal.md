# 提案：初始化 Koa 后端基础工程

## 概要

使用 Koa、TypeScript、REST、Prisma、Redis 和一个示例 `user` 模块，为当前仓库建立第一版后端基础工程。目标是先搭出一个偏生产可用、便于后续扩展的单体后端骨架，再在此基础上逐步补充真实业务。

## 背景

当前仓库还是空的。在真正确定业务方向之前，需要先建立一套稳定的后端基础设施，至少应包含：

- 可运行的 HTTP 服务
- 清晰的模块边界
- 一致的 API 与错误响应约定
- 数据库与 Redis 的接入点
- 本地开发与容器化运行方式
- 基于 spec 的后续变更管理方式

## 目标

- 搭建 Koa + TypeScript 后端骨架
- 采用 REST 作为初始 API 形态
- 集成 Prisma 作为数据库访问层
- 集成 Redis 作为缓存或会话基础设施
- 提供一个 `user` 示例模块，演示标准请求链路
- 提供 Docker 本地开发支持
- 增加基础测试，覆盖底座和示例模块
- 将稳定约定与本次变更记录统一沉淀到 `openspec/`

## 非目标

- 本次不实现鉴权
- 本次不设计完整缓存策略
- 本次不承诺最终业务领域模型，只提供示例模块
- 本次不拆分微服务

## 预期结果

完成后，仓库将具备一套可运行的后端基础工程，并且在 `openspec/` 中拥有清晰的架构设计、任务清单和长期稳定约定，方便后续继续扩展。
